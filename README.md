# InterviewIQ AI

**Multimodal Interview Performance Analyzer**

InterviewIQ AI analyzes mock interview recordings across speech, body
language, and answer quality, then generates a scored, downloadable
performance report. Unlike most mock-interview tools that only ask
questions, InterviewIQ AI tells you *how* you answered, not just *what*.

Runs entirely locally — speech transcription, computer vision, and LLM
evaluation all run on your own machine with **zero API costs** (Whisper +
MediaPipe + a local LLM via [Ollama](https://ollama.com)).

---

## What it does

Upload a video of yourself answering an interview question, and get:

- **Transcript** of your answer (Whisper, run locally)
- **Filler word count** (um, uh, like, you know, etc.)
- **Speaking speed (WPM)** compared against an ideal range
- **Pause detection** — frequency and average length
- **Eye contact %** — combines gaze direction with head pose, so a head
  turn isn't mistaken for "still looking at the camera"
- **Head pose breakdown** — facing camera / looking down / up / left / right
- **Smile frequency**
- **AI feedback on your answer** — strengths, weaknesses, and a rewritten
  improved version, generated by a local LLM (Llama 3.1 via Ollama)
- **Resume-aware evaluation** (optional) — upload your resume alongside
  your answer, and the tool flags specific resume claims (named projects,
  skills, achievements) that you didn't substantiate when you had a
  natural opportunity to
- **Combined scoring** — Confidence, Communication, Body Language,
  Technical, and an Overall score out of 100
- **Downloadable PDF report**

All of this is wrapped in a Streamlit dashboard with tabs for transcript,
speech metrics, body language, AI feedback, and resume check.

## Tech stack

| Layer | Tool |
|---|---|
| Speech-to-text | Whisper (`medium` model, local) |
| Computer vision | OpenCV, MediaPipe FaceMesh |
| Head pose estimation | OpenCV `solvePnP` |
| LLM evaluation | Llama 3.1, run locally via Ollama |
| PDF text extraction (resume) | pdfplumber |
| Video/audio processing | MoviePy |
| UI | Streamlit |
| PDF report generation | ReportLab |

No paid APIs, no API keys, no usage costs. Everything runs on your own
hardware.

## Setup

**Requirements:** Python 3.11, [Ollama](https://ollama.com) installed locally

```bash
git clone https://github.com/<your-username>/interviewiq-ai.git
cd interviewiq-ai

python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate

pip install -r requirements.txt
```

Pull the local LLM model (one-time, ~4.7GB download):
```bash
ollama pull llama3.1
```

No API keys, no `.env` file, no billing setup required.

---

## Run

```bash
streamlit run streamlit_app.py
```

Opens at `http://localhost:8501`. Upload a short video of yourself
answering an interview question, optionally upload your resume, and click
**Analyze interview**.

Note: a full analysis can take 1-3 minutes, since transcription, computer
vision, and LLM evaluation are all running locally rather than on
cloud infrastructure. A progress bar shows which stage is running.

## Known limitations (V1)

- Smile detection and the body-language score weights are heuristic and
  calibrated against a small number of test clips, not a large validated
  dataset — treat scores as directional feedback, not precise measurement.
- Local LLM evaluation (Llama 3.1, 8B) is good at qualitative feedback but
  less reliable at fine-grained numeric rating than larger frontier models.
  Resume-claim matching includes a code-level verification step to catch
  some hallucinated matches, but isn't foolproof.
- Eye contact and head pose currently process the video in separate passes;
  this works but duplicates some video decoding work and could be merged
  into a single pass for efficiency.
- Designed and tested for single-question, single-take video clips
  (roughly 15 seconds to a few minutes), not long-form or multi-question
  recordings.

## Explicitly out of scope for V1

- Live/real-time interview mode
- AI avatars or voice cloning
- Interview scheduling
- A backing database

Keeping V1 tightly scoped was intentional — the goal was a complete,
polished tool rather than an open-ended platform.
